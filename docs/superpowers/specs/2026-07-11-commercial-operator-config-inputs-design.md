# 商業版操作設定與輸入選擇器設計

## 目標

把商業版原本固定的 500 萬操作護欄改為操作者可設定、跨三頁共用且可保存的操作規則；同時讓單股研究可從既有股票清單選擇代號，讓組合健檢可從本機選擇 CSV 檔案。

## 已採用方案

採用「共用設定模組 + 各頁共用設定編輯器」：

- `operator_policy.js` 負責預設值、數值正規化、驗證、衍生金額與儲存格式。
- 新增 `operator_policy_ui.js`，在三頁掛載同一種設定編輯器並發出設定變更回呼。
- 設定存入瀏覽器 `localStorage`；若儲存空間不可用，仍在目前頁面使用已套用值並顯示提示。
- 股票選擇器合併 `/api/watchlist/symbols`、`/api/decision-tracking` 與 `/api/reports` 的 ticker，去重後顯示；仍保留手動輸入。
- CSV 以 `<input type="file">` 選擇本機 `.csv`，瀏覽器只讀文字並填入原有 textarea；實際分析仍使用既有 API。

未採用的方案：只在單一頁面保留設定，會造成三頁試算基準不一致；新增後端帳戶設定 API，超出本次單機操作需求且增加權限與資料遷移成本。

## 操作規則

預設值仍為：

- 操作資金：NT$5,000,000。
- 現金保留：20%。
- 單一持股上限：15%。
- 單筆最大風險：1%。

操作者可修改：

- 操作資金：正數，最低 NT$100,000。
- 現金保留：0–95%。
- 單一持股上限：1–100%。
- 單筆最大風險：0.1–100%。

按「套用設定」後才更新三頁試算；「恢復預設」回到上述預設值。輸入錯誤時不覆寫現有設定，並在設定區顯示可讀錯誤。設定摘要顯示實際金額與百分比，不再出現「固定基準」文案。

## 三頁資料流

### 今日決策

頁面載入保存設定，摘要顯示目前操作資金、現金保留金額、單一持股上限金額與單筆最大風險金額。修改設定後立即重畫摘要；決策 API 與工作排序不變。

### 單股研究

頁面載入股票選項；下拉選擇會填入代號，但仍需按唯一主要動作「更新股票快照」才呼叫快照 API。部位試算必須使用目前設定，而非模組預設值；設定改變後，既有快照立即重新計算股數、投入金額與最大損失。

### 組合健檢

選擇 CSV 後顯示檔名並把內容填入 textarea，操作者可檢查或修改後再分析。`weight` CSV 使用目前設定的操作資金換算；`market_value` CSV 延續使用檔案實際總額。現金門檻、單股上限、減碼金額與健康度均使用目前設定。

## 錯誤處理

- localStorage 內容缺欄位、非數字或超出範圍時回退預設值，不讓無效值進入試算。
- 股票清單任一資料源失敗時，以其他成功來源繼續；全部失敗仍保留手動輸入。
- CSV 不是 `.csv`、無法讀取或為空檔時顯示錯誤，不送出分析 API。
- 設定、股票選擇與 CSV 選擇都是輔助輸入，不新增無作用按鈕。

## 驗證要求

- 純函式測試覆蓋自訂設定、無效值回退、序列化與衍生金額。
- 三頁結構測試覆蓋設定編輯器掛載點、股票 `<select>`、CSV file input 與動態文字。
- 瀏覽器測試覆蓋：改為 1,000 萬後三頁保持一致；股票下拉可選並載入；CSV 檔案選擇後可分析。
- 維持每頁一個主要 CTA、鍵盤操作、44 px 點擊目標與 375/768/1280 無水平捲動。

## 非目標

- 不同步到雲端帳號、不新增後端設定資料表。
- 不代替券商持倉同步。
- 不自動提交股票快照或組合分析；操作者仍需確認後按主要動作。
