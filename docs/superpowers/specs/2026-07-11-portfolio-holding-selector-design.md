# 組合持股選擇器設計

## 目標

讓操作者在組合健檢頁直接從追蹤股票、既有報告與常用股票清單選擇持股，輸入權重後加入或更新組合，並可移除；既有 CSV 選檔、貼上、`weight` 與 `market_value` 流程保持可用。

## 選定方案

採用「持股建立器同步 CSV」：

- 股票清單抽成共用 `ticker_options.js`，單股研究與組合健檢使用同一組 `/api/watchlist/symbols`、`/api/decision-tracking`、`/api/reports` 資料來源與去重規則。
- 組合頁新增股票 `<select>`、權重輸入與次要動作「加入／更新持股」。
- 加入後更新原有 textarea 的 CSV 文字；同一 ticker 再加入會更新權重，不產生重複列。
- 目前選取的 `weight` 持股顯示為可移除清單，移除後同步刪除 CSV 列。
- `Cash` 為固定選項，讓操作者可設定現金權重。

未採用「完全改成持股表格」方案，因為會重做 CSV 驗證、`market_value` 與匯入流程；未採用「只把代號填入 textarea」方案，因為缺少可觀察的加入、更新與移除結果。

## 操作規則

1. 操作者從股票清單選擇 ticker。
2. 輸入 0–100 的權重百分比。
3. 按「加入／更新持股」：
   - CSV 為 `weight` 或 `weight_pct` 格式時，新增或更新該 ticker。
   - CSV 只有 `market_value` 時，不自行換算權重；顯示提示並保留原內容。
4. 持股清單顯示 ticker 與權重，每列提供「移除」。
5. textarea、CSV 檔案或持股建立器任一來源更新內容後，都重新同步可見持股清單。

## 共用股票清單

共用模組合併並去重：

- `/api/watchlist/symbols?q=&limit=25` 的常用清單。
- `/api/decision-tracking` 的追蹤股票。
- `/api/reports?limit=100` 的既有報告。

任一來源失敗時使用其他成功來源；全部失敗時仍保留 `Cash` 與既有 CSV ticker，單股研究仍可手動輸入。

## 錯誤與資料界線

- 未選股票或權重無效時，不修改 CSV，錯誤顯示在持股建立器內。
- 權重允許 0–100；總權重是否超過 100 仍由組合分析結果與現有風險檢查處理。
- 不從股票 API 猜測產業或國家；新列只寫入 ticker 與 weight，其他既有欄位留空。
- 只操作瀏覽器中的 CSV 草稿，不新增後端持股儲存或券商同步。

## 驗證要求

- 純函式測試覆蓋新增、更新、移除、保留額外欄位與拒絕 `market_value` 格式。
- 結構測試覆蓋股票 select、權重 input、加入按鈕、持股清單與共用股票來源模組。
- 瀏覽器測試覆蓋從三類來源選擇、更新同 ticker、不重複、移除、同步 textarea 與送出組合分析。
- 維持每頁一個主要 CTA；持股加入與移除使用次要動作並都有可見結果。
