# 商業版三頁：500 萬資金操作員設計

日期：2026-07-11
狀態：使用者授權直接實作

## 摘要

商業版保留「今日決策、單股研究、組合健檢」三個網址，但由目前的極簡任務頁升級為中密度操作台。目標使用者是管理新台幣 500 萬可投資資金、每天需要決定先看什麼、部位能下多少、風險是否超標，以及下一步如何調整的單一操作員。

新版不恢復舊版的數十個同級模組與假動作。資訊密度增加，但每頁仍維持一個主要資料更新動作，其他互動只做可驗證的篩選、分頁、試算或導覽。

## 已知資料能力

現有 API 已可提供：

- `GET /api/decision-tracking`：建議、初始/最新價格、追蹤報酬、3/6/12 月目標、信心、目標價差、資料時效與需重跑狀態。
- `GET /api/stocks/{ticker}/snapshot`：現價、目標價、估值、分析師共識、營收/獲利成長、利潤率、現金流、資產負債、技術均線、動能、52 週位置、事件與資料品質。
- `POST /api/watchlist/portfolio/risk`：持股權重、產業與國家曝險、最大部位、投資論點缺口及風險旗標。

本次不新增後端投資建議 API。資金金額、部位上限與風險額度均由透明的前端操作規則計算，並與 API 研究結論分開標示。

## 操作員基準

預設操作資金為 `NT$5,000,000`，採用以下可解釋規則：

| 規則 | 預設值 | 金額 |
| --- | ---: | ---: |
| 現金保留 | 20% | NT$1,000,000 |
| 可部署資金 | 80% | NT$4,000,000 |
| 單一持股上限 | 15% | NT$750,000 |
| 單筆最大風險 | 1% | NT$50,000 |

這些數字是操作護欄，不是投資承諾。介面固定顯示「500 萬操作規則試算」，避免被誤認為 AI 報告或後端推薦。

## 方案比較

### 方案 A：恢復舊版完整儀表板

優點是資訊最多，缺點是重新引入 8 個以上分頁、上百個控制與大量無後端支援的動作，會重現使用者原先不知道怎麼用的問題。

### 方案 B：三個中密度操作台（採用）

每頁保留單一核心資料流，在答案卡下增加操作員真正需要的金額、風險、估值與證據區。使用 3–5 個可理解的分頁承載細節，不讓所有模組同時展開。

### 方案 C：逐步精靈

優點是最容易引導新手，缺點是操作員無法快速比較股票、部位與組合狀態，也會破壞現有深連結與熟練操作流程。

## 共用資訊架構

三頁共用以下層級：

1. 頁首導覽與「500 萬操作台」資金標籤。
2. 操作摘要：先回答今天、這檔股票或整體組合最重要的問題。
3. 操作護欄：現金、單一持股上限與單筆風險。
4. 主要資料更新動作。
5. 核心資料表或指標。
6. 漸進揭露分頁。
7. 資料來源、時間、成功與錯誤狀態。

所有金額使用 tabular figures；正負狀態同時使用文字與顏色；互動目標至少 44 px；桌面不超過四欄，手機轉為單欄或可讀資料列。

## 頁面一：今日決策

### 核心問題

500 萬資金今天先處理哪幾檔，原因是什麼？

### 首屏內容

- 操作護欄四格：總資金、現金保留、單一持股上限、單筆最大風險。
- 工作摘要：今日待處理總數、需重跑數、負報酬數、資料更新失敗數。
- 唯一主要 CTA：`檢查最高優先股票`。

### 工作佇列

最多顯示 5 筆，每列包含：

- ticker 與公司。
- 目前建議。
- 最新價格與追蹤報酬。
- 12 月目標價差。
- 信心。
- 優先原因與資料時效。

提供三個有效篩選：`全部`、`需重跑`、`報酬轉弱`。篩選只改變可見列與件數，並以 `aria-live` 回報。每列以連結進入對應單股研究頁。

## 頁面二：單股研究

### 核心問題

這檔股票是否值得占用 500 萬資金中的一個部位，最多能下多少？

### 首屏內容

- ticker 輸入與唯一主要 CTA：`更新股票快照`。
- 結論、現價、目標價、上行空間、資料信心與下一事件。
- `500 萬部位試算`：
  - 預設進場價使用 API 現價。
  - 預設停損參考優先使用 6M 均線；無資料時使用現價下方 8%。
  - 風險部位 = 單筆最大風險 / 每股風險。
  - 最終建議股數同時受單一持股上限限制。
  - 顯示預估投入、占總資金比、每股風險、最大損失與目標潛在報酬。

試算輸入可調整進場價與停損價，修改後立即更新可見數字，不送出後端 mutation。

### 五個分頁

1. `操作計畫`：部位、停損、目標、風險報酬比與資料限制。
2. `估值`：PE、Forward PE、PB、PS、分析師目標與上行空間。
3. `基本面`：營收成長、毛利率、營業利益率、ROE、FCF、現金與負債。
4. `事件`：財報、除息與其他日期，顯示距今天數。
5. `技術`：3M/6M/12M 均線、52 週位置、回撤與 1M/3M/1Y 動能。

API 失敗時保留上一次成功內容，來源區標示錯誤；沒有成功內容時顯示可重試空態。

## 頁面三：組合健檢

### 核心問題

500 萬目前實際配置在哪裡，風險超標多少，應調整多少金額？

### 輸入與主要動作

CSV 保留 `ticker`，以及 `weight` 或 `market_value`。唯一主要 CTA 為 `分析 500 萬組合`。輸入 `weight` 時以 500 萬換算金額；輸入 `market_value` 時以實際總額計算並顯示與 500 萬基準的差異。

### 首屏內容

- 總資金、已配置、現金、最大部位、最大國家曝險。
- 組合健康度與最多 3 項調整建議。
- 建議必須包含金額，例如「2330.TW 由 22% 降至 15%，減少 NT$350,000」。

### 四個分頁

1. `資金配置`：ticker、權重、金額、距 15% 上限的差額。
2. `曝險`：產業與國家權重，以排序資料列呈現，不只使用圖形。
3. `論點`：失效與缺少投資論點的 ticker。
4. `調整清單`：依優先序列出減碼、補現金、分散市場與補論點的具體金額或行動。

本次不恢復無真實資料支援的利率、匯率、籌碼壓力情境按鈕，也不提供看似可執行但不會保存的再平衡單。

## 錯誤與真實資料界線

- 正式頁面不得填入假快照或 fallback portfolio。
- 資金規則衍生值使用 `data-derived-from="operator-policy"` 標示。
- API 結論與操作規則試算在視覺上分成兩個區塊。
- 無法取得必要 API 欄位時顯示 `資料不足`，不以 0 取代未知值。
- CSV 錯誤直接指出列號與修正方法，焦點回到輸入欄。

## 驗證要求

- 三頁各自只有一個主要 CTA，且所有按鈕都可觀察到結果。
- 1280 × 720 首屏可看到操作摘要與主要 CTA；次級資料可在首屏以下。
- 375、768、1280 寬度無水平捲動。
- 鍵盤可操作所有篩選與分頁，focus ring 可見。
- 決策 API 空白/錯誤、股票 API 錯誤、組合 CSV/API 錯誤均有回歸測試。
- 部位試算測試涵蓋風險上限、部位上限、無均線 fallback 與無效價格。
- 金額換算測試涵蓋 `weight`、`market_value`、現金、超額部位與國家曝險。
- 完成後以真實 8080 runtime 截取三頁成功狀態並檢查主控台錯誤。

## 非目標

- 不下單、不連券商、不保存交易。
- 不新增後端情境模擬或再平衡 API。
- 不把營運維護、provider SLA 或內部診斷搬回商業版。
- 不以固定 500 萬試算取代使用者實際券商帳戶資料；本次只建立清楚的操作基準。
